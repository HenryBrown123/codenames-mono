# Transaction Handler Pattern - Codenames Backend

## Overview

The transaction handler pattern ensures **atomicity** for database operations - all operations succeed together or all fail together (rollback).

---

## Core Implementation

### Transaction Handler (`backend/src/common/data-access/transaction-handler.ts`)

```typescript
import type { Kysely, Transaction } from "kysely";
import type { DB } from "../db/db.types";

export type DbContext = Kysely<DB>;
export type TransactionContext = Transaction<DB>;

/**
 * Creates a transactional handler that executes operations within database transactions
 */
export const createTransactionalHandler = <TOperations>(
  db: DbContext,
  createOperations: (trx: TransactionContext) => TOperations,
): TransactionalHandler<TOperations> => {
  return async <TResult>(
    operation: (ops: TOperations) => Promise<TResult>,
  ): Promise<TResult> => {
    return await db.transaction().execute(async (trx) => {
      const operations = createOperations(trx);
      return await operation(operations);
    });
  };
};

export type TransactionalHandler<TOperations> = <TResult>(
  operation: (ops: TOperations) => Promise<TResult>,
) => Promise<TResult>;
```

## Operations Factory Pattern

Define what operations are available within transactions:

```typescript
// backend/src/lobby/lobby-actions.ts
export const lobbyOperations = (trx: TransactionContext) => ({
  // Player operations
  addPlayers: playersRepository.addPlayers(trx),
  removePlayer: playersRepository.removePlayer(trx),
  modifyPlayers: playersRepository.modifyPlayers(trx),

  // Game operations
  updateGameStatus: gamesRepository.updateGameStatus(trx),

  // Round operations
  createRound: newRoundActions.createNextRound(
    roundsRepository.createNewRound(trx),
  ),
  assignPlayerRoles: assignRolesActions.assignRolesRandomly(
    playerRolesRepository.assignPlayerRoles(trx),
    (gameId) => playersRepository.getRoleHistory(trx)(gameId, "CODEMASTER"),
  ),
  dealCards: dealCardsActions.dealCardsToRound(
    cardsRepository.getRandomWords(trx),
    cardsRepository.replaceCards(trx),
    gameEventsRepository.createEvent(trx),
  ),
  startRound: startRoundActions.startCurrentRound(
    roundsRepository.updateRoundStatus(trx),
    turnRepository.createTurn(trx),
  ),
});

export type LobbyOperations = ReturnType<typeof lobbyOperations>;
```

## Usage in Service Layer

```typescript
export const addPlayersService = (dependencies: ServiceDependencies) => {
  return async (gameId: string, userId: number, playersToAdd: PlayerAddData[]) => {
    // 1. Fetch state for validation (OUTSIDE transaction)
    const lobby = await dependencies.getLobbyState(gameId, userId);

    if (!lobby) {
      throw new Error("Game not found");
    }

    // 2. Validate business rules
    if (lobby.status !== "LOBBY") {
      throw new Error("Cannot add players");
    }

    // 3. Execute ALL writes in ONE transaction
    const result = await dependencies.lobbyHandler(async (ops) => {
      const newPlayers = await ops.addPlayers(playersToAdd);
      await ops.updateGameStatus(gameId, "READY");
      return { newPlayers };
    });

    // 4. Emit events AFTER transaction succeeds
    GameEventsEmitter.playersAdded(gameId, result.newPlayers);

    return result;
  };
};
```

## Key Patterns

### ✅ DO: Fetch state BEFORE transaction
```typescript
const lobby = await getLobbyState(gameId, userId);
if (lobby.status !== "LOBBY") throw new Error("Invalid state");

await lobbyHandler(async (ops) => {
  await ops.addPlayers(...);
});
```

### ✅ DO: All writes in ONE transaction
```typescript
await lobbyHandler(async (ops) => {
  await ops.createRound(gameId);
  await ops.dealCards(roundId);
  await ops.updateGameStatus(gameId, "IN_PROGRESS");
  // All succeed or all rollback
});
```

### ✅ DO: Emit events AFTER transaction
```typescript
const result = await lobbyHandler(async (ops) => {
  return await ops.createPlayer(data);
});

GameEventsEmitter.playerJoined(gameId, result.id);
```

### ❌ DON'T: Emit events INSIDE transaction
```typescript
await lobbyHandler(async (ops) => {
  const player = await ops.createPlayer(data);
  GameEventsEmitter.playerJoined(gameId, player.id); // BAD!
  await ops.doSomethingElse(); // If fails, event already sent!
});
```

## Benefits

1. **Atomicity**: All operations succeed or all fail
2. **Type Safety**: Operations object fully typed
3. **Testability**: Easy to mock
4. **Consistency**: Same pattern everywhere
5. **Automatic Rollback**: Kysely handles it
6. **Isolation**: Prevents race conditions

## Example: Start Game Transaction

```typescript
const result = await dependencies.lobbyHandler(async (ops) => {
  // Step 1: Update game status
  await ops.updateGameStatus(lobby._id, "IN_PROGRESS");

  // Step 2: Create first round
  const round = await ops.createRound({ gameId: lobby._id, roundNumber: 1 });

  // Step 3: Assign roles
  const roles = assignRolesRandomly(lobby.players);
  await ops.assignPlayerRoles(round._id, roles);

  // Step 4: Deal cards
  const cards = await ops.dealCards(round._id, lobby.teams);

  // Step 5: Create first turn
  const turn = await ops.createTurn({ roundId: round._id, teamId: lobby.teams[0]._id });

  // If ANY fail, ALL rollback!
  return { round, cards, turn };
});
```

## Summary

**Transaction Handler = Atomic Operations**

1. Create operations factory: `(trx) => ({ ...operations })`
2. Create handler: `createTransactionalHandler(db, operations)`
3. Use in service: `await handler(async (ops) => { ... })`
4. Automatic rollback on error
5. Emit events AFTER success

This ensures **data integrity**!
